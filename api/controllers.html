Controllers¶

Contains the abstract interface for sending commands back to a vehicle interface.

class openxc.controllers.base.Controller¶

A Controller is a physical vehicle interface that accepts commands to be send back to the vehicle. This class is abstract, and implemtnations of the interface must define at least the write_bytes, version, or reset methods.

reset()¶

Reset the vehicle interface.

version()¶

Request and return the version of the vehicle interface.

write(name, value, event)¶

Format the given signal name and value into an OpenXC write request and write it out to the controller interface as bytes, ending with a character.

write_bytes(data)¶

Write the bytes in data out to the controller interface.

write_raw(message_id, data, *args)¶

Format the given CAN ID and data into a JSON message and write it out to the controller interface as bytes, ending with a character.
